Authentication

Login, token management, and authentication flows

Note: Authentication endpoints are public and do not require Bearer tokens. After successful login, use the returned token in the Authorization: Bearer {token} header for subsequent requests.
POST /login
Login to the system and receive authentication token. Token expires after 30 minutes.
Public Endpoint: No authentication required
Request Body 
{
    "email": "user@example.com",
    "password": "your_password"
}
Success Response (200) 
{
    "message": "Login successful",
    "token": "1|abcdefghijklmnopqrstuvwxyz1234567890",
    "user": {
        "hash": "abc123...",
        "name": "John Doe",
        "email": "user@example.com",
        "roles": ["SuperAdmin"],
        "permissions": ["ApplicationsView", "ApplicationsCreate", ...]
    }
}
Error Responses 
// 401 - Invalid Credentials
{
    "message": "Invalid credentials"
}

// 422 - Validation Error
{
    "message": "The email field is required.",
    "errors": {
        "email": ["The email field is required."],
        "password": ["The password field is required."]
    }
}
POST /logout
Logout from the system and invalidate the current authentication token.
Authentication Required: Bearer Token
Request Headers 
Authorization: Bearer 1|abcdefghijklmnopqrstuvwxyz1234567890
Success Response (200) 
{
    "message": "Logged out successfully"
}
Error Responses 
// 401 - Unauthorized
{
    "message": "Unauthenticated."
}

Token Management

  • Token Expiration: Tokens expire after 30 minutes of inactivity
  • Token Format: Sanctum Bearer token (e.g., "1|abcd1234...")
  • Storage: Store token securely (e.g., secure HTTP-only cookie or secure storage)
  • Usage: Include in Authorization header as Bearer {token}

Common Authentication Errors

  • 401 Unauthenticated: Token is missing, invalid, or expired
  • 403 Forbidden: Valid token but insufficient permissions for the requested resource
  • 422 Validation Error: Invalid request body format or missing required fields
