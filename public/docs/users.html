Users

User management and role assignments

Role Required: SuperAdmin only
POST /users/{user}/addLocationAdmin/{location}
Assign a user as LocationAdmin for a specific location.
Roles: SuperAdmin
URL Parameters 
{user}     - User hash or ID
{location} - Location hash or ID
Request Headers 
Authorization: Bearer {token}
Success Response (200) 
{
    "message": "User assigned as LocationAdmin for the location successfully"
}
Error Responses 
// 404 - Not Found
{
    "message": "User not found"
}

// 404 - Not Found
{
    "message": "Location not found"
}

// 403 - Forbidden
{
    "message": "Unauthorized"
}

User Roles

  • SuperAdmin: Full access to all system resources and features
  • LocationAdmin: Can manage locations they are assigned to
  • UserAdmin: Can manage users and access assigned locations

Permissions System

Each role has specific permissions that control access to resources:

  • View Permissions: ApplicationsView, LocationsView, UsersView
  • Create Permissions: ApplicationsCreate, LocationsCreate, UsersCreate
  • Update Permissions: ApplicationsUpdate, LocationsUpdate, UsersUpdate
  • Delete Permissions: ApplicationsDelete, LocationsDelete, UsersDelete
